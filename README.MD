#### 1.1 Register + OTP Verification

**Register Flow:**

![Register](docs/auth%20-%20register.png)

```
[Register Page]
User fills form (name, email, password) ‚Üí submit
        ‚Üì
BE: validate input (Zod)
        ‚Üì
BE: check email exists?
  ‚Üí isVerified: true  ‚Üí error "Email already registered"
  ‚Üí isVerified: false ‚Üí delete old account, create new
        ‚Üì
BE: save user (isVerified: false) + generate 6-digit OTP + send email (Nodemailer)
        ‚Üì
FE: redirect to [OTP Verification Page] (pass email via router state)
```

**OTP Verification Flow:**

![OTP Verify](docs/auth%20-%20OTP%20verify.png)

```
[OTP Verification Page]
User inputs OTP code received via email
        ‚Üì
BE: validate OTP (check code + expiry 15 min)
  ‚Üí expired ‚Üí error "OTP expired, please resend"
  ‚Üí invalid ‚Üí error "Invalid OTP"
  ‚Üí valid   ‚Üí mark isVerified: true, clear OTP
        ‚Üì
FE: redirect to [Login Page]
```

**Edge Cases:**
- Email already registered & verified ‚Üí reject with error
- Email registered but unverified ‚Üí replace account & resend OTP
- OTP expired ‚Üí user must click resend (not re-register)
- Resend OTP ‚Üí BE enforces 60s cooldown (rate-limited), regenerate OTP & resend email
- Brute force OTP ‚Üí rate limiting on `/verify-otp` endpoint

**BE Tasks:**
- [x] User model (`name`, `email`, `password`, `role`, `isVerified`, `otp`, `otpExpires`, `otpLastSentAt`)
- [x] Zod validation schema untuk register, verify OTP, resend OTP
- [x] Email service config (Nodemailer SMTP setup)
- [x] Template email OTP (konten email yang dikirim ke user)
- [x] `POST /api/auth/register` ‚Äî validasi input, cek email, buat user, generate OTP, kirim email
- [x] `POST /api/auth/verify-otp` ‚Äî cek OTP + expiry, tandai akun sebagai verified
- [x] `POST /api/auth/resend-otp` ‚Äî enforce cooldown 60s, generate OTP baru, kirim ulang email
- [x] Rate limit pada `/verify-otp` & `/resend-otp`

**FE Tasks:**
- [x] Zod schema validasi register (nama, email, password, konfirmasi password)
- [x] API service `src/api/auth.ts` ‚Äî fungsi `register()`, `verifyOtp()`, `resendOtp()`
- [x] `/auth/register` ‚Äî halaman form pendaftaran (VeeValidate + Zod)
- [x] `/auth/verify-otp` ‚Äî halaman input OTP (6-digit, resend countdown 60s)
- [x] Handle error response dari BE (email sudah terdaftar, OTP invalid/expired)
- [x] Loading state saat submit form (`isPending` dari TanStack Query `useMutation`)

#### 1.2 Login (Single Device + Token Strategy)

**Rule: 1 account = 1 active device at a time (Web or Mobile)**

**Token Strategy:**
| Token         | Expiry         | Keterangan                                      |
|---------------|----------------|-------------------------------------------------|
| Access Token  | 15 menit       | Dipakai untuk request ke API                    |
| Refresh Token | 1 hari (fixed) | Dipakai untuk minta access token baru. Tidak diperpanjang meski aktif. Setelah 1 hari harus login ulang. |

![Login](docs/auth%20-%20login.png)

**Flow:**
```
[Login Page]
User inputs email + password + deviceId ‚Üí submit
        ‚Üì
BE: validate input (Zod)
        ‚Üì
BE: check credentials + isVerified
        ‚Üì
BE: check active Session for this user
  ‚Üí no session                      ‚Üí buat session baru
  ‚Üí session exists
      ‚Üí refresh token expired        ‚Üí hapus session lama, buat session baru (auto)
      ‚Üí same deviceId               ‚Üí hapus session lama, buat session baru, return token baru
      ‚Üí different device            ‚Üí reject: "Account is active on another device. Please logout first."
        ‚Üì
BE: return access token (15m) + refresh token (1 hari fixed)
        ‚Üì
FE: simpan access token di Pinia (memory), refresh token di localStorage
        ‚Üì
FE: redirect to [Dashboard]

--- saat access token expired (15 menit) ---

FE: kirim refresh token ke BE (silent, di background)
        ‚Üì
BE: cek refresh token valid + belum expired + deviceId sama
        ‚Üì
BE: return access token baru (15m)
        ‚Üì
FE: lanjut pakai app, user tidak sadar

--- setelah 1 hari ---

Refresh token expired ‚Üí FE paksa logout ‚Üí redirect ke login
```

**Device ID:**
- Web: UUID generated on first visit, stored in `localStorage`
- Mobile: native device ID (`expo-device` / `react-native-device-info`)
- Sent in every login request as `deviceId` field

**Session Model (`sessions` collection):**
```
userId            ‚Üí ref to User
deviceId          ‚Üí string (unique per device)
refreshToken      ‚Üí string (hashed dengan bcrypt)
refreshExpiredAt  ‚Üí Date (fixed 1 hari sejak login)
createdAt         ‚Üí Date
```

**Edge Cases:**
- No active session ‚Üí login allowed
- Same device, refresh token still valid ‚Üí perbarui access token
- Different device, refresh token still valid ‚Üí reject dengan error
- Different device, refresh token expired ‚Üí hapus session lama otomatis, login diizinkan
- Refresh token expired ‚Üí paksa logout, redirect ke login

**BE Tasks:**
- [x] Session model (`userId`, `deviceId`, `refreshToken`, `refreshExpiredAt`, `createdAt`)
- [x] Zod validation schema untuk login
- [x] `POST /api/auth/login` ‚Äî validasi kredensial, cek session, hash refresh token, return access token + refresh token
- [x] `POST /api/auth/refresh` ‚Äî validasi refresh token, cek session, return access token baru
- [x] `POST /api/auth/logout` ‚Äî hapus session berdasarkan deviceId
- [x] `GET /api/auth/me` ‚Äî return data user yang sedang login (butuh access token)
- [x] Auth middleware ‚Äî verifikasi access token + cek session aktif di DB
- [x] Rate limit pada `/login`

**FE Tasks:**
- [x] Zod schema validasi login (email, password)
- [x] Generate UUID deviceId saat pertama kali buka app, simpan di localStorage (`useDeviceId` composable)
- [x] API service `src/api/auth.ts` ‚Äî fungsi `login()`, `refresh()`, `logout()`, `getMe()`
- [x] `/auth/login` ‚Äî halaman form login (VeeValidate + Zod)
- [x] Pinia authStore ‚Äî simpan access token di memory, refresh token + user data + deviceId di localStorage
- [x] Load user data via `getMe()` saat app pertama dibuka (jika token masih valid)
- [x] Axios interceptor ‚Äî auto silent refresh saat access token expired (401)
- [x] Handle error "akun aktif di perangkat lain" dengan pesan yang jelas
- [x] Loading state saat submit form (`isPending` dari TanStack Query `useMutation`)

#### 1.3 Forgot Password

**Forgot Password Flow:**

![Forgot Password](docs/auth%20-%20forgot%20password.png)

```
[Forgot Password Page]
User inputs email ‚Üí submit
        ‚Üì
BE: check email exists + generate OTP + send email
  (selalu return 200 meski email tidak ada ‚Äî cegah email enumeration)
        ‚Üì
FE: redirect to [OTP Verification Page] (reset mode, pass email via router state)
        ‚Üì
User inputs OTP ‚Üí BE validate OTP + expiry
        ‚Üì
FE: redirect to [Reset Password Page] (pass email + otp via router state)
```

**Reset Password Flow:**

![New Password](docs/auth%20-%20new%20password.png)

```
[Reset Password Page]
User inputs new password + confirm password ‚Üí submit
        ‚Üì
BE: re-validate OTP (cegah skip halaman verify)
        ‚Üì
BE: update password + hapus semua session aktif (force logout semua device)
        ‚Üì
FE: tampilkan success state ‚Üí redirect to [Login Page]
```

**BE Tasks:**
- [x] Zod validation schema untuk forgot password, verify reset OTP, reset password
- [x] `POST /api/auth/forgot-password` ‚Äî cek email terdaftar, generate OTP, kirim email
- [x] `POST /api/auth/verify-reset-otp` ‚Äî validasi OTP untuk reset password
- [x] `POST /api/auth/reset-password` ‚Äî validasi password baru (min length, konfirmasi), update password, hapus semua session aktif
- [x] Rate limit pada `/forgot-password`

**FE Tasks:**
- [x] Zod schema validasi (email, OTP, password baru + konfirmasi password)
- [x] API service `src/api/auth.ts` ‚Äî fungsi `forgotPassword()`, `verifyResetOtp()`, `resetPassword()`
- [x] `/auth/forgot-password` ‚Äî halaman input email
- [x] `/auth/verify-otp` ‚Äî dipakai ulang dalam mode reset (bukan halaman baru, dibedakan via `history.state.mode`)
- [x] `/auth/reset-password` ‚Äî halaman input password baru + konfirmasi, success state dengan redirect otomatis
- [x] Handle error response dari BE
- [x] Loading state saat submit form (`isPending` dari TanStack Query `useMutation`)

## API Endpoints

### Auth

| Method | Endpoint                    | Description                                         | Auth |
|--------|-----------------------------|-----------------------------------------------------|------|
| GET    | /api/health                 | Health check                                        | -    |
| POST   | /api/auth/register          | Register + send OTP to email                        | -    |
| POST   | /api/auth/verify-otp        | Verify OTP (account activation)                     | -    |
| POST   | /api/auth/resend-otp        | Resend OTP (60s cooldown)                           | -    |
| POST   | /api/auth/login             | Login + device check, return access & refresh token | -    |
| POST   | /api/auth/refresh           | Silent refresh ‚Äî return new access token            | -    |
| POST   | /api/auth/logout            | Logout, delete session                              | JWT  |
| GET    | /api/auth/me                | Get current logged-in user data                     | JWT  |
| POST   | /api/auth/forgot-password   | Send OTP for password reset                         | -    |
| POST   | /api/auth/verify-reset-otp  | Verify OTP for password reset                       | -    |
| POST   | /api/auth/reset-password    | Set new password + delete all active sessions       | -    |

> More endpoints will be added per phase.
