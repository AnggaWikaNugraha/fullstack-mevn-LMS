#### 1.1 Register + OTP Verification

**Register Flow:**

![Register](docs/auth%20-%20register.png)

```
[Register Page]
User fills form (name, email, password) ‚Üí submit
        ‚Üì
BE: validate input (Zod)
        ‚Üì
BE: check if email exists?
  ‚Üí isVerified: true  ‚Üí error "Email already registered"
  ‚Üí isVerified: false ‚Üí delete old account, create new
        ‚Üì
BE: save user (isVerified: false) + generate 6-digit OTP + send email (Nodemailer)
        ‚Üì
FE: redirect to [OTP Verification Page] (pass email via router state)
```

**OTP Verification Flow:**

![OTP Verify](docs/auth%20-%20OTP%20verify.png)

```
[OTP Verification Page]
User inputs OTP code received via email
        ‚Üì
BE: validate OTP (check code + expiry 15 min)
  ‚Üí expired ‚Üí error "OTP expired, please resend"
  ‚Üí invalid ‚Üí error "Invalid OTP"
  ‚Üí valid   ‚Üí mark isVerified: true, clear OTP
        ‚Üì
FE: redirect to [Login Page]
```

**Edge Cases:**
- Email already registered and verified ‚Üí reject with error
- Email registered but unverified ‚Üí replace account and resend OTP
- OTP expired ‚Üí user must click resend (not re-register)
- Resend OTP ‚Üí BE enforces 60s cooldown (rate-limited), regenerate OTP and resend email
- Brute force OTP ‚Üí rate limiting on `/verify-otp` endpoint

**BE Tasks:**
- [x] User model (`name`, `email`, `password`, `role`, `isVerified`, `otp`, `otpExpires`, `otpLastSentAt`)
- [x] Zod validation schema for register, verify OTP, resend OTP
- [x] Email service config (Nodemailer SMTP setup)
- [x] OTP email template (HTML email sent to user)
- [x] `POST /api/auth/register` ‚Äî validate input, check email, create user, generate OTP, send email
- [x] `POST /api/auth/verify-otp` ‚Äî check OTP + expiry, mark account as verified
- [x] `POST /api/auth/resend-otp` ‚Äî enforce 60s cooldown, generate new OTP, resend email
- [x] Rate limit on `/verify-otp` & `/resend-otp`

**FE Tasks:**
- [x] Zod schema for register validation (name, email, password, confirm password)
- [x] API service `src/api/auth.ts` ‚Äî `register()`, `verifyOtp()`, `resendOtp()`
- [x] `/auth/register` ‚Äî registration form page (VeeValidate + Zod)
- [x] `/auth/verify-otp` ‚Äî OTP input page (6-digit, 60s resend countdown)
- [x] Handle error responses from BE (email already registered, OTP invalid/expired)
- [x] Loading state on form submit (`isPending` from TanStack Query `useMutation`)

#### 1.2 Login (Single Device + Token Strategy)

**Rule: 1 account = 1 active device at a time (Web or Mobile)**

**Token Strategy:**
| Token         | Expiry         | Description                                                                                                  |
|---------------|----------------|--------------------------------------------------------------------------------------------------------------|
| Access Token  | 15 minutes     | Used for all API requests                                                                                    |
| Refresh Token | 1 day (fixed)  | Used to request a new access token. Not extended on activity. After 1 day the user must log in again.       |

![Login](docs/auth%20-%20login.png)

**Flow:**
```
[Login Page]
User inputs email + password + deviceId ‚Üí submit
        ‚Üì
BE: validate input (Zod)
        ‚Üì
BE: check credentials + isVerified
        ‚Üì
BE: check active session for this user
  ‚Üí no session                      ‚Üí create new session
  ‚Üí session exists
      ‚Üí refresh token expired        ‚Üí delete old session, create new session (auto)
      ‚Üí same deviceId               ‚Üí delete old session, create new session, return new tokens
      ‚Üí different device            ‚Üí reject: "Account is active on another device. Please logout first."
        ‚Üì
BE: return access token (15 min) + refresh token (1 day fixed)
        ‚Üì
FE: store access token in Pinia (memory), refresh token in localStorage
        ‚Üì
FE: redirect to [Dashboard]

--- when access token expires (15 minutes) ---

FE: send refresh token to BE (silent, in background)
        ‚Üì
BE: validate refresh token + not expired + same deviceId
        ‚Üì
BE: return new access token (15 min)
        ‚Üì
FE: continue using the app, user is unaware

--- after 1 day ---

Refresh token expired ‚Üí FE force logout ‚Üí redirect to login
```

**Device ID:**
- Web: UUID generated on first visit, stored in `localStorage`
- Mobile: native device ID (`expo-device` / `react-native-device-info`)
- Sent in every login request as `deviceId` field

**Session Model (`sessions` collection):**
```
userId            ‚Üí ref to User
deviceId          ‚Üí string (unique per device)
refreshToken      ‚Üí string (hashed with bcrypt)
refreshExpiredAt  ‚Üí Date (fixed 1 day from login)
createdAt         ‚Üí Date
```

**Edge Cases:**
- No active session ‚Üí login allowed
- Same device, refresh token still valid ‚Üí issue new access token
- Different device, refresh token still valid ‚Üí reject with error
- Different device, refresh token expired ‚Üí auto-delete old session, login allowed
- Refresh token expired ‚Üí force logout, redirect to login

**BE Tasks:**
- [x] Session model (`userId`, `deviceId`, `refreshToken`, `refreshExpiredAt`, `createdAt`)
- [x] Zod validation schema for login
- [x] `POST /api/auth/login` ‚Äî validate credentials, check session, hash refresh token, return access token + refresh token
- [x] `POST /api/auth/refresh` ‚Äî validate refresh token, check session, return new access token
- [x] `POST /api/auth/logout` ‚Äî delete session by deviceId
- [x] `GET /api/auth/me` ‚Äî return currently logged-in user data (requires access token)
- [x] Auth middleware ‚Äî verify access token + check active session in DB
- [x] Rate limit on `/login`

**FE Tasks:**
- [x] Zod schema for login validation (email, password)
- [x] Generate UUID deviceId on first app open, store in localStorage (`useDeviceId` composable)
- [x] API service `src/api/auth.ts` ‚Äî `login()`, `refresh()`, `logout()`, `getMe()`
- [x] `/auth/login` ‚Äî login form page (VeeValidate + Zod)
- [x] Pinia authStore ‚Äî store access token in memory, refresh token + user data + deviceId in localStorage
- [x] Load user data via `getMe()` on app startup (if token is still valid)
- [x] Axios interceptor ‚Äî auto silent refresh when access token expires (401)
- [x] Handle "account active on another device" error with a clear message
- [x] Loading state on form submit (`isPending` from TanStack Query `useMutation`)

#### 1.3 Forgot Password

**Forgot Password Flow:**

![Forgot Password](docs/auth%20-%20forgot%20password.png)

```
[Forgot Password Page]
User inputs email ‚Üí submit
        ‚Üì
BE: check if email exists + generate OTP + send email
  (always returns 200 even if email not found ‚Äî prevents email enumeration)
        ‚Üì
FE: redirect to [OTP Verification Page] (reset mode, pass email via router state)
        ‚Üì
User inputs OTP ‚Üí BE validate OTP + expiry
        ‚Üì
FE: redirect to [Reset Password Page] (pass email + otp via router state)
```

**Reset Password Flow:**

![New Password](docs/auth%20-%20new%20password.png)

```
[Reset Password Page]
User inputs new password + confirm password ‚Üí submit
        ‚Üì
BE: re-validate OTP (prevents skipping the verify step)
        ‚Üì
BE: update password + delete all active sessions (force logout all devices)
        ‚Üì
FE: show success state ‚Üí redirect to [Login Page]
```

**BE Tasks:**
- [x] Zod validation schema for forgot password, verify reset OTP, reset password
- [x] `POST /api/auth/forgot-password` ‚Äî check email exists, generate OTP, send email
- [x] `POST /api/auth/verify-reset-otp` ‚Äî validate OTP for password reset
- [x] `POST /api/auth/reset-password` ‚Äî validate new password (min length, confirm match), update password, delete all active sessions
- [x] Rate limit on `/forgot-password`

**FE Tasks:**
- [x] Zod schema validation (email, OTP, new password + confirm password)
- [x] API service `src/api/auth.ts` ‚Äî `forgotPassword()`, `verifyResetOtp()`, `resetPassword()`
- [x] `/auth/forgot-password` ‚Äî email input page
- [x] `/auth/verify-otp` ‚Äî reused in reset mode (not a new page, differentiated via `history.state.mode`)
- [x] `/auth/reset-password` ‚Äî new password + confirm password page, success state with auto redirect
- [x] Handle error responses from BE
- [x] Loading state on form submit (`isPending` from TanStack Query `useMutation`)

#### 1.5 Global FE ‚Äî Error Handling

**Error handling rules:**

| Error Type          | Example                         | Handler                                     |
|---------------------|---------------------------------|---------------------------------------------|
| Network error       | No internet, timeout            | Global modal                                |
| Server error (5xx)  | Server crash, maintenance       | Global modal                                |
| Auth error (401)    | Token expired                   | Silent refresh ‚Üí if failed ‚Üí redirect login |
| Validation error (400) | Invalid email format         | Inline in form                              |
| Not found (404)     | Data not found                  | Inline in form                              |

- [x] `src/api/client.ts` ‚Äî Axios interceptor catches network errors & 5xx, triggers errorStore
