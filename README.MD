#### 1.2 Login (Single Device + Token Strategy)

**Rule: 1 account = 1 active device at a time (Web or Mobile)**

**Token Strategy:**
| Token         | Expiry         | Description                                                                                                  |
|---------------|----------------|--------------------------------------------------------------------------------------------------------------|
| Access Token  | 15 minutes     | Used for all API requests                                                                                    |
| Refresh Token | 1 day (fixed)  | Used to request a new access token. Not extended on activity. After 1 day the user must log in again.       |

![Login](docs/auth%20-%20login.png)

**Login Flow:**

```
User inputs email + password ‚Üí submit
          |
          ‚ñº
    POST /auth/login  (+ deviceId in body)
          |
          ‚îú‚îÄ‚Üí Validation error (Zod)         ‚Üí error with field messages
          |
          ‚îú‚îÄ‚Üí Email not found                ‚Üí error "Email or password incorrect"
          |
          ‚îú‚îÄ‚Üí Wrong password                 ‚Üí error "Email or password incorrect"
          |
          ‚îú‚îÄ‚Üí Account not verified           ‚Üí error "Please verify your email first"
          |
          ‚îî‚îÄ‚Üí Valid & verified
                    |
                    ‚ñº
              Check active session
                    |
                    ‚îú‚îÄ‚Üí No session                    ‚Üí create new session
                    |
                    ‚îú‚îÄ‚Üí Same deviceId                 ‚Üí replace session, return new tokens
                    |
                    ‚îú‚îÄ‚Üí Different device, expired     ‚Üí delete old session, create new session
                    |
                    ‚îî‚îÄ‚Üí Different device, still valid ‚Üí error "Account is active on another device"
                              |
                              ‚ñº
                        Generate access token (15 min)
                        Generate refresh token (1 day fixed, hashed in DB)
                        Save session to DB
                              |
                              ‚ñº
                        FE: store access token in Pinia (memory)
                        FE: store refresh token + deviceId in localStorage
                        FE: redirect to [Home]
```

**Silent Refresh Flow:**

```
API request returns 401 (access token expired)
          |
          ‚ñº
    POST /auth/refresh  (refresh token + deviceId)
          |
          ‚îú‚îÄ‚Üí Refresh token invalid/expired  ‚Üí force logout ‚Üí redirect to [Login Page]
          |
          ‚îî‚îÄ‚Üí Valid
                    |
                    ‚ñº
              Return new access token (15 min)
                    |
                    ‚ñº
              FE: retry original request ‚Äî user is unaware
```

**Logout Flow:**

```
User clicks logout
          |
          ‚ñº
    POST /auth/logout  (requires JWT)
          |
          ‚ñº
    Delete session from DB (by deviceId)
          |
          ‚ñº
    FE: clear Pinia store + localStorage
    FE: redirect to [Login Page]
```

**Device ID:**
- Web: UUID generated on first visit, stored in `localStorage`
- Mobile: native device ID (`expo-device` / `react-native-device-info`)
- Sent in every login request as `deviceId` field

**Session Model (`sessions` collection):**
```
userId            ‚Üí ref to User
deviceId          ‚Üí string (unique per device)
refreshToken      ‚Üí string (hashed with bcrypt)
refreshExpiredAt  ‚Üí Date (fixed 1 day from login)
createdAt         ‚Üí Date
```

**Edge Cases:**
- No active session ‚Üí login allowed
- Same device, refresh token still valid ‚Üí issue new access token
- Different device, refresh token still valid ‚Üí reject with error
- Different device, refresh token expired ‚Üí auto-delete old session, login allowed
- Refresh token expired ‚Üí force logout, redirect to login

**BE Tasks:**
- [x] Session model (`userId`, `deviceId`, `refreshToken`, `refreshExpiredAt`, `createdAt`)
- [x] Zod validation schema for login
- [x] `POST /api/auth/login` ‚Äî validate credentials, check session, hash refresh token, return access token + refresh token
- [x] `POST /api/auth/refresh` ‚Äî validate refresh token, check session, return new access token
- [x] `POST /api/auth/logout` ‚Äî delete session by deviceId
- [x] `GET /api/auth/me` ‚Äî return currently logged-in user data (requires access token)
- [x] Auth middleware ‚Äî verify access token + check active session in DB
- [x] Rate limit on `/login`

**FE Tasks:**
- [x] Zod schema for login validation (email, password)
- [x] Generate UUID deviceId on first app open, store in localStorage (`useDeviceId` composable)
- [x] API service `src/api/auth.ts` ‚Äî `login()`, `refresh()`, `logout()`, `getMe()`
- [x] `/auth/login` ‚Äî login form page (VeeValidate + Zod)
- [x] Pinia authStore ‚Äî store access token in memory, refresh token + user data + deviceId in localStorage
- [x] Load user data via `getMe()` on app startup (if token is still valid)
- [x] Axios interceptor ‚Äî auto silent refresh when access token expires (401)

#### 1.3 Login with Google (OAuth 2.0)

**Account Rules:**
- Google user (new email) ‚Üí create account automatically (`isVerified: true`, no password)
- Google email matches existing account ‚Üí **merge**: add `googleId` to existing user, can login both ways
- Google-only user tries email/password login ‚Üí guided error: "Set a password first via Forgot Password" + link to Forgot Password page
- After setting password via Forgot Password ‚Üí account becomes dual-method (Google + email/password)
- Google user uses Forgot Password ‚Üí allowed: OTP sent, can set password, account becomes dual-method

**Google Login Flow (Authorization Code ‚Äî Redirect):**

```
User clicks "Continue with Google"
          |
          ‚ñº
    FE redirect ‚Üí accounts.google.com/o/oauth2/v2/auth
    (user selects account on Google's own page)
          |
          ‚îú‚îÄ‚Üí User cancels ‚Üí Google redirects back with error ‚Üí FE redirect to /auth/login
          |
          ‚îî‚îÄ‚Üí User selects account
                    |
                    ‚ñº
              Google redirects to /auth/google/callback?code=xxx
                    |
                    ‚ñº
              GoogleCallbackView extracts code from URL
                    |
                    ‚ñº
              POST /api/auth/google  { code, deviceId }
                    |
                    ‚îú‚îÄ‚Üí code invalid/expired    ‚Üí error "Google authentication failed"
                    |
                    ‚îî‚îÄ‚Üí code valid
                              |
                              ‚ñº
                        BE exchanges code ‚Üí Google Token endpoint
                        ‚Üí get idToken ‚Üí verify ‚Üí extract:
                        email, name, googleId (sub)
                              |
                              ‚îú‚îÄ‚Üí Email not in DB
                              |         |
                              |         ‚ñº
                              |   Create new user
                              |   (isVerified: true, googleId, no password)
                              |
                              ‚îú‚îÄ‚Üí Email exists, no googleId (merge)
                              |         |
                              |         ‚ñº
                              |   Add googleId to existing user
                              |
                              ‚îî‚îÄ‚Üí Email exists, has googleId
                                        |
                                        ‚ñº
                                  [Session logic ‚Äî same as regular login]
                                  Check active session
                                        |
                                        ‚îú‚îÄ‚Üí Different device, still valid
                                        |   ‚Üí error "Account is active on another device"
                                        |
                                        ‚îî‚îÄ‚Üí No session / same device / expired

                                                  ‚ñº

```
